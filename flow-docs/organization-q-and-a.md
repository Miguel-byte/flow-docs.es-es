# <a name="flow-in-your-organization-qa"></a>El flujo de la organización Q & A
[!INCLUDE [view-pending-approvals](includes/cc-rebrand.md)]
En este tema se describe cómo los usuarios de su organización pueden usar Flow y cómo puede controlar el servicio Flow.

## <a name="signing-up-for-flow"></a>Registro en Flow
### <a name="what-is-microsoft-flow"></a>¿Qué es Microsoft Flow?
Microsoft Flow es un servicio en la nube pública que ayuda a los usuarios y equipos a configurar flujos de trabajo automatizados entre sus aplicaciones y servicios favoritos para sincronizar, obtener notificaciones, recopilar datos y mucho más. 

### <a name="how-do-people-sign-up-for-flow"></a>¿Cómo se suscriben los usuarios a Flow?
Hay dos formas posibles de que los usuarios puedan suscribirse a Flow a través del portal web:

#### <a name="option-1"></a>Opción 1
Cualquier usuario puede registrarse; para ello, vaya a [Flow.Microsoft.com](https://flow.microsoft.com), seleccione **suscribirse gratis**y, después, complete el proceso de registro para fluir a través de [admin.Microsoft.com](https://admin.microsoft.com/Start?sku=flow_free) o [signup.Live.com](https://signup.live.com).

#### <a name="option-2"></a>Opción 2
Cualquier usuario puede registrarse yendo a [Flow.Microsoft.com](https://flow.microsoft.com), seleccionando **iniciar sesión**, iniciando sesión con su trabajo, escuela o correo electrónico personal y aceptando los términos de flujo de uso.    

Cuando un usuario de la organización se suscribe a Flow con la opción 2, se le asignará un Microsoft Flow licencia gratuita automáticamente.

[Regístrese en Flow](sign-up-sign-in.md) incluye más detalles.

### <a name="what-is-the-microsoft-flow-free-plan"></a>¿Cuál es el plan gratuito Microsoft Flow?

El plan gratuito de Microsoft Flow se usa solo con fines de seguimiento. Habilitar o deshabilitar no tiene ningún efecto en la capacidad de un usuario para crear flujos. Si deshabilita el plan gratis de Microsoft Flow, se vuelve a habilitar cuando un usuario inicia sesión. Este es el comportamiento esperado.

### <a name="can-i-block-another-person-from-signing-up-for-flow"></a>¿Puedo bloquear la suscripción de otra persona a Flow?
Microsoft Flow es un servicio en la nube totalmente público y todo el mundo puede registrarse y usarlo para automatizar sus tareas cotidianas. Para usar Microsoft Flow no hay ningún requisito que los usuarios tengan o usen una cuenta de Office 365. Por este motivo, no hay ningún mecanismo en este momento para impedir que otra persona use Flow (como todo el mundo puede, independientemente de su dirección de correo electrónico).

Sin embargo, si una persona se suscribe a Microsoft Flow y usted decide no admitirlos dentro de su organización, no podrá incurrir en ningún tipo de costos a su empresa. Cuando una persona se suscribe a Microsoft Flow, la relación se encuentra entre esa persona y Microsoft, que es como muchos otros servicios en la nube de Microsoft, como Bing, Wunderlist, OneDrive o Outlook.com. El uso de Microsoft Flow por parte de un usuario no implica que el servicio sea proporcionado por su organización.

Por último, si su empresa desea restringir el uso de datos solo de la organización dentro de Microsoft Flow, esto es posible a través de las directivas de prevención de pérdida de datos (DLP).

### <a name="how-can-people-gain-access-to-the-paid-features-of-microsoft-flow"></a>¿Cómo pueden los usuarios obtener acceso a las características de pago de Microsoft Flow?
Los usuarios pueden obtener acceso a las características de pago de Microsoft Flow de tres maneras diferentes:

1. Pueden registrarse individualmente para un plan de flujo 1 o un plan de flujo 2 de prueba de 90 días gratis
2. Puede asignarles una licencia de flujo en el portal de administración de Office 365.
3. Se ha asignado al usuario planes de Office 365 y Dynamics 365 que incluyen el acceso al servicio de flujo. Consulte la [Página de precios de Flow](https://flow.microsoft.com/pricing/) para ver la lista de planes de Office 365 y Dynamics 365 que incluyen funcionalidades de Flow.

### <a name="can-i-block-another-person-from-using-the-paid-features-of-flow"></a>¿Se puede impedir que otra persona use las características de pago de Flow?
Cualquier persona puede probar las características de pago de Microsoft Flow durante 90 días y no incurrir en ningún costo. Sin embargo, puede administrar por completo la asignación de las licencias de pago perpetua dentro de la organización a través del portal de administración de Office 365.

Al igual que con las ofertas gratuitas, si una persona decide suscribirse a la versión de prueba que es una relación directa entre el individuo y Microsoft, no necesariamente aprobado por su empresa.

## <a name="administration-of-flow"></a>Administración de Flow
### <a name="why-has-the-flow-icon-appeared-in-the-office-365-app-launcher"></a>¿Por qué aparece el icono de flujo en el iniciador de aplicaciones de Office 365?
Tal como se anunció en agosto, Microsoft Flow es ahora una parte fundamental del conjunto de Office 365. Tres meses después de este anuncio Microsoft Flow se habilitó como un servicio como parte de todas las SKU existentes de Office 365. A medida que los usuarios de todo el mundo pueden usar Microsoft Flow, ha aparecido en el iniciador de aplicaciones para ellos.

Consulte la sección siguiente si desea quitar el icono de flujo del iniciador de aplicaciones de forma predeterminada.

### <a name="how-do-i-remove-microsoft-flow-from-the-app-launcher-for-my-organization"></a>¿Cómo quitar Microsoft Flow del iniciador de aplicaciones de mi organización?
Si a un usuario se le asignó un plan de flujo 1 o una licencia de plan 2 de Flow, puede realizar los pasos siguientes para quitar la licencia de flujo para ese usuario, lo que eliminará el icono de flujo del iniciador de aplicaciones:

1. Vaya al [portal de administración de Office 365](https://portal.microsoftonline.com/).
2. En la barra de navegación izquierda, seleccione **usuarios**y, a continuación, seleccione **usuarios activos**.
3. Busque el usuario para el que desea quitar la licencia y seleccione su nombre.
4. En el panel de detalles del usuario, en la sección **licencias de producto** , seleccione **Editar**.
5. Busque la licencia llamada **Microsoft Flow plan 1** o **Microsoft Flow plan 2**, establezca el control de alternancia en **desactivado** y, a continuación, seleccione **Guardar**.
   
   ![](./media/organization-q-and-a/remove-license.png)

Si un usuario tiene acceso a Flow a través de su licencia de plan de Office 365 y Dynamics 365, puede deshabilitar su acceso a las características adicionales incluidas en este plan realizando los pasos siguientes:

1. Vaya al [portal de administración de Office 365](https://portal.microsoftonline.com/).
2. En la barra de navegación izquierda, seleccione **usuarios**y, a continuación, seleccione **usuarios activos**.
3. Busque el usuario al que desea quitar el acceso y, a continuación, seleccione su nombre.
4. En el panel de detalles del usuario, en la sección **licencias de producto** , seleccione **Editar**.
5. Expanda la licencia de Office 365 o Dynamics 365 del usuario, deshabilite el acceso al servicio denominado **Flow para Office 365** o **flow for Dynamics 365** y seleccione **Guardar**.
   
   ![](./media/organization-q-and-a/remove-service-plan.png)

También es posible la eliminación masiva de licencias mediante PowerShell. Consulte [eliminación de licencias de cuentas de usuario con Office 365 PowerShell](https://technet.microsoft.com/library/dn771774.aspx) para obtener un ejemplo detallado.   Por último, puede encontrar más instrucciones sobre la eliminación masiva de servicios dentro de una licencia en [deshabilitar el acceso a los servicios con Office 365 PowerShell](https://technet.microsoft.com/library/dn771769.aspx).

La eliminación del servicio o la licencia de Flow para un usuario de la organización producirá la eliminación del icono de flujo de las siguientes ubicaciones para ese usuario:

1. [Office.com](https://office.com)
   
   ![](./media/organization-q-and-a/office-home.png)
2. Iniciador de aplicaciones de Office 365
   
   ![](./media/organization-q-and-a/office-waffle.png)

Tenga en cuenta que esto solo quitará el mosaico Flow de forma predeterminada. Un usuario puede seguir usando Microsoft Flow como individuo.

### <a name="why-did-10000-licenses-for-microsoft-flow-show-up-in-my-office-365-tenant"></a>¿Por qué las licencias 10.000 para Microsoft Flow aparecen en mi inquilino de Office 365?
Cualquier persona puede probar Microsoft Flow plan 1 o 2 durante 90 días, y estas licencias de prueba representan la capacidad disponible para nuevos usuarios de Flow en el inquilino. Estas licencias no se cobran. En concreto, hay dos razones posibles por las que puede ver que en el portal de administración de Office 365 se muestran licencias de capacidad 10.000 (evaluación):

1. Si al menos un usuario del inquilino ha participado en la versión preliminar pública de Flow que abarca de abril de 2016 a octubre de 2016, verá 10.000 licencias con la etiqueta "Microsoft PowerApps y flujos de lógica"
   
    ![](./media/organization-q-and-a/licenses2.png)
2. Si al menos un usuario del inquilino se ha suscrito a una prueba de Flow plan 2 mediante la **opción 1** de la suscripción de prueba que se describe en la sección [¿cómo se suscriben los usuarios a PowerApps?](#how-do-people-sign-up-for-flow) , verá las licencias 10.000 con la etiqueta "flujo de & de Microsoft Power apps"
   
    ![](./media/organization-q-and-a/licenses1.png)

Puede asignar licencias adicionales a los usuarios a través del portal de administración de Office 365, pero tenga en cuenta que se trata de licencias de prueba para Microsoft Flow plan 2 y expirarán después de que se asignen 90 días a un usuario.

### <a name="is-this-free-will-i-be-charged-for-these-licenses"></a>¿Es gratis? ¿Se me cobrará por estas licencias?
Ningún usuario puede incurrir en ningún costo a su organización sin su consentimiento expreso, por lo que ni las licencias gratuitas ni de evaluación pueden provocar cargos en la organización. Además, tampoco usan cuotas, como las cuotas de ejecución.

### <a name="i-removed-the-microsoft-flow-free-license-and-users-can-still-access-flow"></a>He quitado la licencia gratuita de Microsoft Flow y los usuarios todavía pueden acceder a Flow?
La licencia gratuita de Microsoft Flow se incluye solo con fines de seguimiento. Como se describe en la primera sección, no es posible impedir que otra persona use Microsoft Flow para fines individuales. Por lo tanto, la presencia de una licencia gratuita de Microsoft Flow no concede ni quita ninguna funcionalidad.

### <a name="why-cant-i-see-all-flow-licenses-in-the-office-365-admin-portal"></a>¿Por qué no puedo ver todas las licencias de flujo en el portal de administración de Office 365?
Los usuarios pueden usar Microsoft Flow como individuos o como parte de su organización. Las licencias en el nivel de organización siempre estarán visibles en el portal de Office 365. Sin embargo, si un usuario se suscribe a una versión de prueba como un individuo que no está administrado por el administrador de Office 365 y no se mostrará en el portal.

### <a name="how-does-an-individual-find-out-what-plan-they-are-on"></a>¿Cómo averigua un individuo en qué plan están?
Cualquier usuario puede ver el plan que tienen visitando la página de precios de Flow en [https://flow.microsoft.com/pricing](https://flow.microsoft.com/pricing). El plan o la evaluación en que se encuentran actualmente se mostrarán allí.

### <a name="will-microsoft-flow-sign-up-impact-the-identities-in-my-organization"></a>¿Microsoft Flow Suscríbase afectará a las identidades de mi organización?
Si su organización ya tiene un entorno de Office 365 existente y todos los usuarios de su organización tienen cuentas de Office 365, la administración de identidades no se verá afectada.

Si su organización ya tiene un entorno de Office 365 existente pero no todos los usuarios de su organización tienen cuentas de Office 365, crearemos un usuario en el inquilino y asignaremos licencias en función de la dirección de correo electrónico profesional o educativa del usuario. Esto significa que el número de usuarios que administra en un momento determinado crecerá a medida que los usuarios de su organización se suscriban al servicio.

Si su organización no tiene un entorno de Office 365 conectado al dominio de correo electrónico, no hay ningún cambio en el modo de administrar la identidad. Los usuarios se agregarán a un nuevo directorio de usuarios solo en la nube y tendrá la opción de tomar el control como administrador de inquilinos y administrarlos.

### <a name="a-new-tenant-was-created-by-microsoft-flow-how-do-i-manage-this"></a>Microsoft Flow creó un nuevo inquilino, ¿cómo lo administro?
Si Microsoft Flow creó un nuevo inquilino, puede reclamarlo y administrarlo mediante los siguientes pasos:

1. Únase al inquilino mediante un dominio de correo electrónico que coincida con el dominio del inquilino que desea administrar. Por ejemplo, si Microsoft creó el inquilino de contoso.com, únase al inquilino con una dirección de correo electrónico que termine con @contoso.com.
2. Control de la administración de notificaciones mediante la comprobación de la propiedad del dominio: una vez que se encuentre en el inquilino, puede promoverse a sí mismo al rol de administrador comprobando la propiedad del dominio. Para ello, siga estos pasos:    
   
   1. Vaya a [https://admin.microsoft.com](https://admin.microsoft.com/Start?sku=flow_free).
   2. Seleccione el icono del iniciador de aplicaciones en la parte superior izquierda y elija administrador.
   3. Lea las instrucciones de la página **convertirse en Administrador** y, a continuación, elija **sí, deseo ser administrador**.  
      
       **Nota**: Si esta opción no aparece, ya existe un administrador de Office 365.

### <a name="if-i-have-multiple-domains-can-i-control-the-office-365-tenant-that-users-are-added-to"></a>Si tengo varios dominios, ¿puedo controlar el inquilino de Office 365 al que se agregan los usuarios?
Si no hace nada, se creará un inquilino para cada dominio y subdominio de correo electrónico de usuario.

Si desea que todos los usuarios estén en el mismo inquilino independientemente de sus extensiones de dirección de correo electrónico:  

* Cree un inquilino de destino de antemano o use un inquilino existente. Agregue todos los dominios y subdominios existentes que desee que se consoliden en ese inquilino. A continuación, todos los usuarios con direcciones de correo electrónico que terminen en esos dominios y subdominios se unen automáticamente al inquilino de destino cuando se suscriben.

**Importante**: no hay ningún mecanismo automatizado compatible para trasladar usuarios entre inquilinos una vez que se han creado. Para obtener información sobre cómo agregar dominios a un solo inquilino de Office 365, vea [Agregar usuarios y dominios a office 365](https://support.office.com/article/Add-your-users-and-domain-to-Office-365-ffdb2216-330d-4d73-832b-3e31bcb5b2a7).

### <a name="how-can-i-restrict-my-users-ability-to-access-my-organizations-business-data"></a>¿Cómo puedo restringir la capacidad de los usuarios para acceder a los datos empresariales de mi organización?
Microsoft Flow permite crear zonas de datos para datos empresariales y no empresariales, como se muestra a continuación. Una vez implementadas estas directivas de prevención de pérdida de datos, los usuarios no podrán diseñar ni ejecutar flujos que combinen datos empresariales y no empresariales. Para obtener más información, consulte [directivas de prevención de pérdida de datos (DLP)](prevent-data-loss.md).

  ![](./media/organization-q-and-a/data-loss-prevention-policy.png)
